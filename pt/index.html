Como funciona

Quatro primitivos. Um sistema coerente.

Contexto, memória, hooks e skills — os blocos de construção que fazem o HIVE parecer um time, não um chatbot.

1

Contexto carrega automaticamente

O CLAUDE.md de cada squad define sua persona, escopo e regras. O Claude Code carrega ao navegar para a pasta do squad — sem copiar e colar prompts entre sessões.

2

Memória persiste entre sessões

O STATE.md armazena o estado operacional em três camadas: L1 (agora), L2 (em andamento), L3 (backlog). Decisões e gotchas ficam registrados indefinidamente.

3

Hooks automatizam o trabalho de bastidor

9 hooks gerenciam session branches, roteamento de squads, atualizações de estado e buscas de incidentes — sem você precisar gerenciar nada disso manualmente.

4

Skills são comandos reutilizáveis

Invoque com /nome-da-skill. Vem com /hive-setup, /open-squad, /status, /close-squad — e você escreve as suas para sua empresa.

Início Rápido

Rodando em menos de 10 minutos

Clone, abra o Claude Code, rode o onboarding. Esse é o processo de instalação completo.

terminal 
# 1. Clone o repositório
git clone https://github.com/felipeluissalgueiro/hive.git minha-empresa
cd minha-empresa

# 2. Instale o Claude Code (pule se já tiver)
curl -fsSL https://claude.ai/install.sh | sh

# 3. Abra e execute o onboarding guiado (~10 min)
claude
/hive-setup

# Comece a trabalhar com um squad
/open-squad commercial

# Veja o que está acontecendo em todos os squads
