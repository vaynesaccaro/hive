# Onboarding Playbook

## Overview
```
Kickoff → Setup → Training → Go-Live
```
Target time-to-value: **___ days**

---

## Phase 1: Kickoff
- **Timeline:** Day 0–3
- **Owner:** CS Lead
- **Activities:**
  - [ ] Intro call — align goals, stakeholders, timeline
  - [ ] Share onboarding checklist with client
  - [ ] Confirm technical contacts and access requirements
  - [ ] Set up shared Slack / communication channel
- **Success Criteria:** Client aligned on scope and timeline; kickoff doc sent

---

## Phase 2: Setup
- **Timeline:** Day 4–14
- **Owner:** CS + Solutions Engineer
- **Activities:**
  - [ ] Provision account / workspace
  - [ ] Configure integrations (list: ___)
  - [ ] Migrate or import existing data
  - [ ] Complete environment checklist
- **Success Criteria:** System live in test/staging; no blocking errors

---

## Phase 3: Training
- **Timeline:** Day 15–21
- **Owner:** CS
- **Activities:**
  - [ ] Admin training session (record and share)
  - [ ] End-user training session
  - [ ] Deliver documentation / help resources
  - [ ] Q&A async support window open
- **Success Criteria:** Champion and at least 2 end users can perform core workflow independently

---

## Phase 4: Go-Live
- **Timeline:** Day 22–30
- **Owner:** CS
- **Activities:**
  - [ ] Confirm go-live date with client
  - [ ] Monitor first 48h for errors or blockers
  - [ ] Send go-live summary + what to expect next
  - [ ] Schedule 30-day check-in
- **Success Criteria:** Client running in production; health score green; NPS ≥ _

---

## Escalation
- Blocked > 3 business days → CS Manager notified
- Client unresponsive > 5 business days → escalate to account owner
