# OKR Framework

## Format

```
Objective: [Qualitative, inspiring, directional — no metrics]
  KR1: [Measurable outcome — baseline → target by date]
  KR2: [Measurable outcome — baseline → target by date]
  KR3: [Measurable outcome — baseline → target by date]
```

### Rules
- 1–3 Objectives per team per quarter
- 2–4 Key Results per Objective
- KRs must be outcomes, not activities ("increase X from Y to Z", not "run weekly reviews")
- No copy-pasting last quarter's OKRs without deliberate update

---

## Example

```
Objective: Become the default tool for ops teams in mid-market
  KR1: Increase trial-to-paid conversion from 18% to 28%
  KR2: Close 15 new logos in the $20k–$50k ARR band
  KR3: NPS from active accounts ≥ 45
```

---

## Quarterly Cadence

| Month | Activity |
|-------|---------|
| Last 2 weeks of quarter | Draft next quarter's OKRs |
| First week of new quarter | Finalize + align with team |
| Week 4 of each month | Mid-quarter check-in (score + adjust) |
| Last week of quarter | Final score + retrospective |

---

## Review Ritual

**Monthly check-in (30 min):**
1. Score each KR (0.0–1.0)
2. Flag blockers — what's off track and why
3. Agree on adjustments or escalations

**Quarterly retrospective:**
1. Final scores
2. What drove results?
3. What should we start / stop / continue?
4. Feed learnings into next quarter's OKRs

---

## Grading Scale

| Score | Interpretation |
|-------|---------------|
| 0.9–1.0 | Crushed it (or set the bar too low) |
| 0.7–0.89 | Strong — on target |
| 0.4–0.69 | Partial progress — needs attention |
| 0.0–0.39 | Missed — root cause required |

> Consistently scoring 1.0 means KRs aren't ambitious enough.
